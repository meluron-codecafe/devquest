Comparing Pitch Trackers

praat-parselmouth; pyreaper; librosa; pyworld; libtsm;

Quest¶

  1. What are the tools available to perform pitch tracking?
  2. How long do they take to compute pitch?
  3. How do they perform in their default settings?
  4. What user-controls do they provide?
  5. How to choose the right one for your task?

Tools Available for Pitch Tracking¶

Below is a list of tools available to extract pitch of an audio signal.

  • Praat with Parselmouth as a python wrapper.
  • Pyworld
  • Librosa
  • Pyreaper

Runtime Measurement¶

We first try to test how different pitch trackers perform in their default settings. It is an uncontrolled experiment where we take an example audio without its ground truth f0 contour for comparison. The idea is to make some visual observations.

import parselmouth as pm
import pyworld as pw
import pyreaper as pr
import librosa as lb

import numpy as np
import modusa as ms
import time
import pandas as pd



def get_f0_contour(y, sr, lib):
    """
    Compute the F0 contour using different
    python libraries in their default
    settings.

    Parameters
    ----------
    y: ndarray
        Audio signal to compute the F0 contour of.
    sr: int
        Sampling rate of the audio signal.
    lib: str
        Which library to use for computing F0 contour.
        "pm" for parselmouth, "pw" for pyworld, "lr" for librosa, "pr" for pyreaper

    Returns
    -------
    ndarray
        F0 contour.
    ndarray
        time stamps.
    """

    # Parselmouth
    def f0_pm():
        snd = pm.Sound(values=y, sampling_frequency=sr)
        pitch_obj = snd.to_pitch_ac()
        f0 = pitch_obj.selected_array['frequency']
        t = pitch_obj.ts()

        return f0, t

    # Pyworld
    def f0_pw():
        _f0, t = pw.dio(y, sr)
        f0 = pw.stonemask(y, _f0, t, sr) # This is for refining pitch

        return f0, t

    # Librosa
    def f0_lr():
        f0, _, _ = lb.pyin(y=y, sr=sr, fmin=lb.note_to_hz('C2'), fmax=lb.note_to_hz('C7')) # Recommended parameter choice (check docstr)
        t = lb.times_like(f0, sr=sr)

        return f0, t

    # Pyreaper
    def f0_pr():
        y_int = (y * 32767).astype(np.int16)
        _, _, t, f0, _ = pr.reaper(x=y_int, fs=sr)

        return f0, t


    dispatcher = {
        "pm": f0_pm,
        "pw": f0_pw,
        "lr": f0_lr,
        "pr": f0_pr
    }

    if lib not in dispatcher:
        raise ValueError(f"Invalid lib keyword, choose from {dispatcher.keys}")

    # Print the time taken
    tic = time.time()
    f0, f0_t = dispatcher[lib]()
    toc = time.time()
    runtime = toc - tic # Seconds
    
    return f0, f0_t, runtime

# Load a sample audio signal
def test_time_complexity(dur):
    """
    To test the how long different tools
    take to compute the pitch of an audio
    signal of duration `dur` seconds

    Parameters
    ----------
    dur: float
        - Duration (sec) of the audio to load for testing the tools.

    Returns
    -------
    tuple:
        Duration of the loaded audio signal, Execution time for Praat Parselmouth, Pyworld Dio, Librosa pYIN, Pyreaper in seconds.
    """
    
    y, sr, title = ms.load("../../../data/fsm-snd/Crocodile 1.mp3", sr=16000, clip=dur)
    ann = ms.load_ann("../../../data/fsm-snd/Crocodile 1-words.txt") # Loading word-level annotation (manually annotated)

    f0_pm, t_pm, pm_runtime = get_f0_contour(y=y, sr=sr, lib="pm") # Parselmouth
    f0_pw, t_pw, pw_runtime = get_f0_contour(y=y, sr=sr, lib="pw") # Pyworld
    f0_lr, t_lr, lr_runtime = get_f0_contour(y=y, sr=sr, lib="lr") # Librosa
    f0_pr, t_pr, pr_runtime = get_f0_contour(y=y, sr=sr, lib="pr") # Pyreaper
    
    return (y.size / sr, pm_runtime, pw_runtime, lr_runtime, pr_runtime)

Performance in Default Settings¶

While it's important to fine-tune the parameters of these tools depending on the type of audio signals, for now, let's try them with the default settings to see how they perform. We've already computed the result for an audio signal above so let’s examine the contour and make some observations.

Please note that the observations below are specific to the audio file that we happen to choose randomly:

  1. Praat-parselmouth gives us promising results compared to other tools.
  2. Pyreaper and Librosa are more sensitive towards giving non-zero values in unvoiced regions.
  3. Pyreaper is resistant towards big sudden jumps and thus prevent octave jumps around 2.2 sec.
